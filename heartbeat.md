## UPDATE RULES

These rules govern how this file must be maintained. They apply to every agent session that reads this file.

1. **Always read this file first** before generating any content plan or content output.
2. **Always update this file last** after completing a content task or weekly plan session.
3. **Never delete previous entries** — append to logs, do not overwrite them.
4. **Keep entries concise** — one to two lines per log entry maximum.
5. **Update `current_focus` and `next_suggested_action`** at the end of every session based on what was done and what logically comes next.
6. **If a milestone is reached** (award, demo launch, release), mark it in `milestone_log` and update the relevant game's `last_content_date`.
7. **This file can be updated manually** by the user at any time — agent must respect manual edits and not overwrite them.
